# Парсер российских сервисов предоставляющих доступ к картам

English version is
[here](
    https://github.com/EvgenyFreelance/Russian-maps-Parser
)

Работа этого парсера заключается в сборе инофрмации с сайтов перечисленных выше.Также сам парсер имеет гибкую настройку(API) и может через него менять свое поведение и возвращать нужный результат в зависимости от предпочтений пользователя.
## Содержание проекта
Содержит 4 основных файла для работы, а именно:
- main.py
- gui.py
- export_data.py
- backend_of_gui.py

Пойдем по порядку:

<h2>
    <p align='center'>main.py</p>
</h2>

этот файл содержит основную логику работы парсера:
``` python
Parser()
```
```Parser()``` - класс, некое представление парсера. Имеет методы:
- `scrolling_down()`
- `parse()`
- `get_result()`
- `stop()`
___
```scrolling_down(self,page,selector)```- Прокручивает страницу вниз до заданного элемента. В качестве параметров принимает 2 значения ```page``` и ```selector```.

- ```page``` - Страница на которую будем скроллить.
- ```selector``` - Элемент до которого будем скроллить.

```parse(self)``` - Запускает парсер с заданными параметрами.

```get_result(self))``` - Возвращает полученные данные.

```stop()``` - Останавливет работу парсера(с последующий возможностю применить метод ```get_result()```).

<h2>
    <p align='center'>gui.py</p>
</h2>

этот файл содержит **GUI** через которое пользователь может взаимодействовать с парсером в реальном времени:

``` python
window()
```

```window()``` - функция запускающая окно(с GUI) через которое можно контролировать работу парсера

<h2>
    <p align='center'>export_data.py</p>
</h2>

В этом файле в основном находятся функции, заточенные на сохранении данных в те или иные форматы.

Основные функции:
- `to_sqlite()`
- `to_excel()`
___
- ```to_sqlite(data,path)``` - принимает 2 аргумента data и path и при использовании сохраняет данные в формат .db по указанному пути.
  - data - данные которые нужно сохранить .
  - path - путь куда сохранять файл.

- ```to_excel(data,path)``` - принимает 2 аргумента data и path и при использовании сохраняет данные в формат .xlsx по указанному пути.
  - data - данные куда нужно сохранить .
  - path - путь куда сохранять файл.
  
<h2>
    <p align='center'>backend_of_gui.py</p>
</h2>

В этом же файле содержется backend логика для GUI.Здес находятся функции которые распределяют действия по процесса и потокам во избежания GIL. Он также был создан чтобы излишне не делать громоздким файл gui.py.

Основные компоненты:
- `start_parsing()`
- `nested_process()`
- `saving()`
- `get_txt()`
- `get_json()`
___

- `start_parsing(start_bool_sm,pipe,txt_path,json_path**kwargs,)` - Это функция которая запускает `nested_process()` в отдельном процессе и передает ему все свои аргументы

 - `nested_process(bool_sm,pipe,txt_path,json_path,**kwargs)` функция с 2-мя потоками, 1 из которых запускает сам парсер, а другой его останавливает с сохранением данных.
   - bool_sm - переменная которую постоянно просматривает 2 поток, чтобы закончить работу парсера
   - pipe - переменная ,через которую общаются процессы и через неё в конечном итоге возвращаются полученные данные
   - txt_path - .txt файл с запросами, по которым парсер будет работать
   - json_path - .json файл с конфигурацией парсера(к примеру задание прокси).
- `saving(data,export_pieces)` - функция сохраняющая данные по всем заданными экспортам указанным в GUI.
  - data - данные которые будем сохранять
  - export_pieces - список с состовляющими путя а именно:  ['Путь до папки','Название файла','Формат']
- `get_txt(path)` - функция читающая .txt файл по заданному пути и возвращающая массив строк
  - path - путь до файла
- `get_json(path)` - функция читающая .json файл по заданному пути и возвращающая словарь, который потом может передаватсья в парсер через **.
  - path - путь до файла
